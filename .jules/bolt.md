## 2026-03-02 - NumPy vectorized aggregation over Python list appends
**Learning:** In tight sub-stepping loops within Python numerical simulations (like PARCEM morphodynamic models), using Python dictionaries to store lists of values (`qs_step_vals[name].append(...)`) inside a time loop causes substantial overhead due to continuous reallocations and slow lookups. Using pre-initialized NumPy arrays to sum values vectorially (`qs_step_sum += Qs[target_idx_arr]`) and computing the average once per outer loop dramatically reduces interpreter overhead and speeds up the model.
**Action:** When tracking multiple flux locations or variables in inner simulation loops, always try to index the specific values into a single NumPy array and perform vectorized summation/averaging rather than appending to individual Python lists.
