## 2024-03-14 - Optimize inner loop with pre-allocation and vectorized append
**Learning:** Python operations like appending to dicts iteratively inside a tight numerical loop create massive overhead compared to using an array and `np.mean(..., axis=0)`. Furthermore, allocating arrays inside inner loops using `np.empty_like` or `np.zeros` adds up, so reusing a pre-allocated `out` array gives a measurable speedup. Replacing division by multiplications using inverted values slightly improves performance. Avoiding repeated `np.maximum(0.0, arr)` allocations inside inner loops can also be achieved by using array indexing (e.g. `arr[arr < 0] = 0.0`).
**Action:** Next time working on tight numerical simulation loops in numpy, avoid recreating dicts or calling array allocations inside the loop. Instead, use pre-allocated buffers passed as arguments, convert to vectorized lists/arrays for aggregations, and favor multiplication over division. Keep artifact files clean out of commits.
